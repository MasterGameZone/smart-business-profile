# SMART BUSINESS PROFILE
## Version 0.5 – Session-Based Business Profile

## Standard Instructions

Before making any changes:

1. Read the existing codebase completely.
2. Preserve the current project architecture.
3. Reuse existing components whenever possible.
4. Preserve the current design language.
5. Modify only the files required for this version.
6. Do not refactor unrelated code.
7. Do not add extra features.
8. Do not anticipate future versions.

---

## 1. Objective

Improve the Business Profile Preview by allowing users to freely move between the form and the preview without losing their entered information during the current browser session.

This version improves usability only.

Do NOT permanently save data.

---

## 2. Existing Project Context

Current project includes:

- React + Vite + TypeScript
- Tailwind CSS
- React Router
- Landing Page
- Business Profile Form
- Business Profile Preview
- Temporary in-memory data transfer

Continue working inside the existing project.

Do not restructure the application.

---

## 3. Build

Improve the current data flow.

The user should be able to:

- Fill the form.
- Open the preview.
- Return to Edit.
- Change any field.
- Open the preview again.

All entered information should remain available throughout the current browser session.

Do NOT use:

- Local Storage
- Session Storage
- Database
- Backend

Use React Context (or another appropriate in-memory global state solution) to manage the profile data.

---

## 4. Navigation

Ensure these flows work correctly:

Home

↓

Create Profile

↓

Profile Preview

↓

Back to Edit

↓

Profile Preview

The entered information must remain available during this navigation.
