## 1. Objective

Protect business management features so that only authenticated users can create, edit, and manage business profiles.

Associate each Business Profile with the authenticated user.

---

## 2. Existing Project Context

Current project includes:

- Supabase Authentication
- Business Profile CRUD
- Public Business Profile Pages
- Existing Routing

Continue working inside the existing project.

Do not restructure the application.

---

## 3. Database Changes

Create a new SQL migration.

Modify the existing business_profiles table.

Add:

owner_id

Properties:

- UUID
- Nullable initially
- Foreign Key referencing auth.users(id)

Backfill existing rows as NULL.

Do not delete existing data.

---

## 4. Save Profile

When creating a new profile:

- User must be authenticated.
- Automatically save:
  owner_id = authenticated user's ID.

Do not ask the user to choose an owner.

---

## 5. Protected Routes

Protect the following routes:

- Create Profile
- Edit Profile
- Save Profile

If the user is not logged in:

- Redirect to Login.
- Preserve the originally requested route if practical.

Public profile pages must remain publicly accessible.

---

## 6. Authorization

When editing a profile:

Verify:

owner_id == authenticated user ID

If the profile belongs to another user:

Display:

403 - Access Denied

Do not expose profile editing.

---

## 7. Navigation

When logged in:

Show:

- Dashboard
- Logout

When logged out:

Show:

- Login
- Sign Up

Keep navigation simple.

---

## 8. Row Level Security

Create a SQL migration updating RLS policies.

Rules:

Authenticated users:

- Can create their own profiles.
- Can update only their own profiles.

Anonymous users:

- Can read public business profiles.
- Cannot create or update profiles.
