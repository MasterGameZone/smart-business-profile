## 1. Objective

Build Version 0.6 of Smart Business Profile.

The objective is to automatically save the business profile locally so users do not lose their information after refreshing the browser.

This is the first persistence feature.

Use browser Local Storage only.

---

## 2. Existing Project Context

Current project includes:

- React + Vite + TypeScript
- Tailwind CSS
- React Router
- Business Profile Form
- Business Profile Preview
- React Context for temporary state

Continue working inside the existing project.

Do not restructure the application.

---

## 3. Build

Implement automatic Local Storage support.

When the user updates any field:

- Save the latest profile data automatically.
- Restore the data automatically when the application loads.
- The user should not have to click a Save button.

The restore should happen silently.

---

## 4. Logo Handling

Do NOT attempt to store image files in Local Storage.

Store every field except the uploaded logo.

When the application reloads:

- Restore all text information.
- Clear the logo field.
- Display the default placeholder until the user selects the logo again.

---

## 5. Clear Data

Add a new button on the Create Profile page:

Clear Form

When clicked:

- Clear all form fields.
- Remove the saved Local Storage data.
- Reset the Context state.
- Navigate back to the empty form.

Ask for confirmation before clearing.

---

## 6. State Management

Continue using the existing React Context.

Local Storage should synchronize with Context.

Context remains the application's working state.

Local Storage acts only as persistent storage.
