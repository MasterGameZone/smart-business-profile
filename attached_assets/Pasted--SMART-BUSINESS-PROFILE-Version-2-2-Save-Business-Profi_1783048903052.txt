# SMART BUSINESS PROFILE
## Version 2.2 – Save Business Profile to Supabase

## Standard Instructions

Before making any changes:

1. Read the existing codebase completely.
2. Preserve the current project architecture.
3. Reuse existing components whenever possible.
4. Modify only the files required for this version.
5. Do not refactor unrelated code.
6. Do not add extra features.
7. Use the existing Supabase connection.
8. Use the existing business_profiles table.
9. Stop immediately after completing Version 2.2.

---

## 1. Objective

Replace Local Storage with Supabase for saving Business Profiles.

The user should now save their profile into the database instead of browser Local Storage.

Do NOT implement profile retrieval.

Do NOT implement editing.

Do NOT implement authentication.

---

## 2. Existing Project Context

Current project includes:

- React + Vite + TypeScript
- Tailwind CSS
- React Router
- Business Profile Form
- Profile Preview
- Supabase connection
- business_profiles table
- QR Code
- Share Profile

Continue working inside the existing project.

Do not restructure the project.

---

## 3. Build

When the user clicks "Save Profile":

- Validate the form.
- Insert a new row into the business_profiles table.
- Show a loading state while saving.
- Prevent duplicate submissions.
- Show a professional success toast after successful save.

Do not redirect the user.

Remain on the current page.

---

## 4. Error Handling

If saving fails:

- Show a user-friendly error toast.
- Do not crash the application.
- Keep the entered data intact.
- Log the detailed error only to the console.

---

## 5. Local Storage

Remove Local Storage as the primary persistence mechanism.

If any Local Storage logic exists for saving profiles:

- Remove it.
- Keep only temporary UI state if absolutely necessary.

The database becomes the single source of truth for profile storage.
