## 1. Objective

Implement real authentication using Supabase Auth.

This version should connect the existing Authentication UI to Supabase.

Users should be able to:

- Create an account
- Login
- Logout
- Receive email verification
- Request password reset

## 3. Sign Up

Connect the existing Sign Up page to Supabase Auth.

Requirements:

- Create account using email and password.
- Validate existing client-side rules before submission.
- Show loading state.
- Prevent duplicate submissions.

If successful:

Display a success message:

"Account created successfully. Please verify your email before logging in."

Do not automatically log the user in.

---

## 4. Login

Connect the Login page.

Requirements:

- Email
- Password

If login succeeds:

- Create a Supabase session.
- Redirect to the Home page.

If login fails:

Display a user-friendly error toast.

Do not expose raw Supabase errors.

---

## 5. Logout

Implement logout functionality.

Requirements:

- Clear the current session.
- Redirect to the Landing Page.
- Display a success toast.

Do not remove any user data.

---

## 6. Forgot Password

Connect the existing Forgot Password page.

Requirements:

- Send a password reset email using Supabase Auth.
- Display a generic success message.

Do not reveal whether an email exists.

---

## 7. Reset Password

Implement password reset.

Requirements:

- Update the password using Supabase Auth.
- Display a success toast.
- Redirect to Login after success.

Reuse the existing Reset Password page.

---

## 8. Authentication State

Create a reusable authentication service.

Monitor authentication state.

Provide helper methods:

- signUp()
- signIn()
- signOut()
- resetPassword()
- updatePassword()
- getCurrentUser()

Use reusable code.

Avoid duplicating Supabase calls.

## 9. Session Persistence

The user should remain logged in after refreshing the page.

Use the default Supabase session persistence.

- Wrong password

- Weak password
- Network error

Display professional user-friendly messages.

Log detailed errors only to the console.
